# 票鉴

浏览器本地运行的发票中奖核对工具。用户分别上传发票图片/PDF和中奖信息图片，应用完成多发票切分、OCR、重复核对、号码匹配、统计与 CSV 导出。

## 运行

```bash
npm install
npm run dev
```

打开 `http://127.0.0.1:5173/`。

生产构建：

```bash
npm run build
npm run preview
```

## 数据与隐私

- 应用不会扫描或读取项目目录中的历史发票。
- 只有用户在两个上传入口中主动选择的文件会参与本次核对。
- OCR、号码解析和匹配在当前浏览器中执行；没有业务后端或数据库。
- 首次识别时 Tesseract.js 会下载中英文 OCR 模型，随后由浏览器缓存。发票文件本身不会因此上传。
- 发票区域检测使用按需加载的 OpenCV.js；没有接入 Agent、VLM 或其他外部推理服务。
- 刷新页面或点击“清除本次数据”会丢弃当前页面中的识别结果。

## 匹配规则

- 优先比较完整发票号码。
- 中奖名单只有尾号时，支持不少于 6 位的尾号比较。
- 号码缺失、OCR 置信度低或出现多条候选时标记为“待确认”，不会直接计为未中奖。
- OCR 结果在匹配前可以人工修改；中奖号码也可以手动增删。

## 多发票与重复处理

- 一张图片或 PDF 单页可以检测多个发票区域；识别前可拖动、缩放、增加或删除区域框。
- 号码清单图片保留整图即可，应用会从多行文字中生成多条发票记录。
- 重复候选综合文件 SHA-256、发票代码/号码和图片感知哈希生成。
- 重复候选不会自动删除；必须选择保留项，或者确认“不是重复”。未处理完不能生成统计。
- 完全相同的文件共享 OCR 结果，批量识别最多使用两个 Worker，并按每页 50 条展示复核结果。
- CSV 会保留被排除的重复记录、来源页码、区域和人工决策，便于审计。

## 识别扩展接口

`src/lib/recognitionProvider.js` 定义了识别 Provider 边界。当前只提供 `local-ocr` 实现，未来可以增加用户主动触发的低置信度 VLM 兜底，但默认流程不会调用大模型。

## 支持格式

- 发票：JPG、PNG、WebP、PDF（逐页识别）。
- 中奖信息：JPG、PNG、WebP。
- 导出：带 UTF-8 BOM 的 CSV，可直接用 Excel 打开。
